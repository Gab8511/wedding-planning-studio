# Privacy

VowSuite Wedding Planning Studio is currently a static browser app. It does not require an account, server, database, or third-party analytics service.

## Local Storage

Wedding plans are stored in the browser's local storage on the device where the app is used. This can include guest details, RSVP notes, vendor information, budgets, task notes, timeline details, and planning preferences entered into the app.

## Data Sharing

The app does not automatically send wedding data to a remote server unless optional cloud sync is configured. Data leaves the browser when a user chooses to export, copy, print, download, push to cloud sync, pull from cloud sync, or otherwise share it.

## Optional Cloud Sync

Cloud sync uses the Supabase project URL, anon public key, and shared sync ID saved in the browser. Do not enter a service role key. Privacy and access control depend on the Supabase row-level security policies configured by the workspace owner.

## Backups

Use the app's export tools to create backups before clearing browser data, changing devices, or resetting the app for a new couple.

## Public Demo

Avoid entering sensitive real-world personal, financial, or contract information into a public demo environment unless you understand where the browser stores that data and how to clear it.

## Future Services

If hosted accounts, email integrations, payments, or public vendor portals are added later, this privacy file should be updated before those features are released.
